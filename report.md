# Отчёт: Контекстный переранкер на основе эмбеддингов

## Abstract

В работе исследуется метод контекстного переранжирования рекомендаций в музыкальном сервисе botify. Базовый рекомендер SasRec-I2I выбирает следующий трек на основе item-to-item соседей, случайно выбирая якорный трек из истории. Предлагаемый подход реализует двухэтапный пайплайн: сначала формируется расширенный пул кандидатов (HSTU user-based рекомендации + I2I соседи), затем кандидаты переранжируются с помощью cosine similarity между PCA-сжатыми эмбеддингами треков и вектором контекста сессии, построенным как взвешенное среднее эмбеддингов прослушанных треков. A/B эксперимент на 30000 пользователей показал значимое улучшение ключевой метрики mean_time_per_session на +49.9%.

## Детали реализации

Метод реализует serving-часть two-tower архитектуры, аналогичной YouTube DNN и DSSM:

**Session tower (query encoder):** Для каждого запроса вычисляется вектор контекста сессии как взвешенное среднее эмбеддингов недавно прослушанных треков. Веса равны доле прослушивания трека (0.0–1.0); скипнутые треки (time < 0.05) исключаются, чтобы негативные сигналы не искажали вектор контекста.

**Candidate tower:** Кандидаты собираются из двух источников — персональные HSTU-рекомендации (до 100 треков) и I2I-соседи для двух лучших якорей из сессии. Каждый кандидат представлен своим PCA-сжатым эмбеддингом.

**Scoring и re-ranking:** Финальный скор = cosine similarity между вектором сессии и эмбеддингом кандидата, с мультипликативным штрафом 0.5^k за артистов, уже встречавшихся k раз в сессии.

**Оптимизации для низкой латентности:** Исходные 4096-мерные LLM-эмбеддинги сжимаются randomized PCA до 64 измерений при старте сервера. Маппинг track -> artist строится в памяти при инициализации, исключая redis-вызовы на этапе scoring.

```
┌─────────────────┐     ┌──────────────┐     ┌──────────────┐
│  История сессии │     │  HSTU recs   │     │ I2I соседи   │
│  (track, time)  │     │ (user-based) │     │ (item-based) │
└────────┬────────┘     └──────┬───────┘     └──────┬───────┘
         │                     └───────┬────────────┘
         V                             V
┌─────────────────┐           ┌──────────────────┐
│ Weighted avg    │           │ Пул кандидатов   │
│ pooling -> PCA  │           │ (без повторов)   │
│ -> normalize    │           └──────┬───────────┘
└────────┬────────┘                  │
         │         cosine sim        │
         └──────────────┬────────────┘
                        V
              ┌──────────────────┐
              │ Artist penalty   │
              │ -> argmax -> rec │
              └──────────────────┘
```

## Результаты A/B эксперимента

Эксперимент HSTU, 30000 эпизодов, seed=31312. Control (C) — SasRec-I2I, Treatment (T1) — Contextual Re-ranker.

| Метрика             | Control (C) | Treatment (T1) |  Effect | Significant |
|---------------------|------------:|---------------:|--------:|:-----------:|
| Суммарное время     |       40.02 |          60.59 |  +51.4% |     Да      |
| Число сессий        |        5.98 |           6.06 |   +1.3% |     Нет     |
| Треков на сессию    |       11.72 |          15.06 |  +28.5% |     Да      |
| Время на сессию     |        6.73 |          10.08 |  +49.9% |     Да      |
| Задержка (мс/запрос)|        0.59 |           0.86 |  +46.1% |     Да      |

Метод показывает значимое улучшение по всем ключевым метрикам: пользователи слушают на 28.5% больше треков за сессию и проводят на 49.9% больше времени. Число сессий статистически не изменилось, что говорит об улучшении качества рекомендаций внутри сессии без влияния на возвращаемость. Рост латентности после оптимизации (PCA, in-memory artist map) составил всего 46% (с 0.59 до 0.86 мс), что приемлемо для продакшена.
