# Отчёт: Контекстный переранкер в рекомендательной системе

## Abstract

В работе исследуется метод контекстного переранжирования рекомендаций в музыкальной системе. Предлагаемый подход объединяет кандидатов из нескольких источников (персональные рекомендации HSTU и item-to-item похожие треки) и переупорядочивает их с использованием косинусного сходства между эмбеддингами треков и вектором текущей сессии пользователя. Контекст сессии строится как взвешенное среднее эмбеддингов недавно прослушанных треков, где веса соответствуют времени прослушивания. Предполагается, что учёт сессионного контекста позволяет улучшить пользовательский опыт по сравнению с базовым I2I-рекомендером.

---

## Детали реализации

Система реализует двухэтапный пайплайн рекомендаций:

1. **Генерация кандидатов**:

   * рекомендации HSTU (user-based)
   * I2I-рекомендации для недавно прослушанных треков

2. **Переранжирование**:

   * вычисление вектора сессии как взвешенного среднего эмбеддингов треков
   * ранжирование кандидатов по косинусному сходству
   * штраф за повторение артистов для повышения разнообразия

Контекст сессии учитывает длительность прослушивания: треки с малым временем (скипы) практически не влияют на итоговый вектор.

**Схема пайплайна:**

```id="pipeline_ru"
История пользователя -> Вектор сессии -> Генерация кандидатов (HSTU + I2I) -> 
-> Переранжирование (cosine similarity) -> Штраф за повтор артистов -> Итоговая рекомендация
```

## Результаты A/B эксперимента

Сравнение проводилось между базовой моделью (C) и предложенным методом (T1).

| Метрика                | Control | Treatment | Effect | Significant |
| ---------------------- | ------: | --------: |-------:|-------------|
| Суммарное время        |   21.55 |     27.39 | +27.1% | Да          |
| Число сессий           |    3.16 |      3.14 | -0.62% | Нет         |
| Треков на сессию       |   11.92 |     14.06 | +18.0% | Да          |
| Время на сессию        |    6.94 |      9.08 | +30.9% | Да          |
| Задержка (latency, мс) |    0.70 |     19.14 | +2620% | Да          |

Метод показывает значимое улучшение пользовательского взаимодействия: пользователи слушают больше треков и проводят больше времени в рамках одной сессии. При этом число сессий не изменилось, что означает отсутствие влияния на частоту возвращения пользователей, но заметное улучшение качества рекомендаций внутри сессии.

Основным недостатком является существенный рост времени ответа системы, что может негативно сказаться на пользовательском опыте и масштабируемости.

## Вывод

Контекстное переранжирование позволяет существенно повысить качество рекомендаций и вовлечённость пользователей за счёт учёта текущего контекста сессии. Однако вычислительная сложность метода приводит к значительному увеличению задержек. Для практического применения необходимо дополнительно оптимизировать модель (например, за счёт сокращения числа кандидатов или использования приближённых методов поиска).
